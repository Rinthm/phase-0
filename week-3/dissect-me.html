Galivanting With Git

Exploration of the command line, version control, git and GitHub.

10/25/15

Hello there! This week I have learned how to do many things in the mysterious realm of the command line. I also picked up some sweet knowledge about using git, version control, and the awesome powers of GitHub! Let me take a few minutes to bestow some of this knowledge upon you!

First, I'll describe some of the terms I just dropped. The "command line" is essentially the environment that allows a user to communicate with their computer - the user enters an input and the computer will produce an output! "Version control" is the process of keeping track of who makes changes to a program, when the changes are made, and the details of each change. "Git" is a popular version control system. "GitHub" is a website where programmers can store thier code. Okay! Now that we have a general idea of what we're looking at, I'll discuss why this is important.

So let's say a team of programmers wants to work on a project together that is located on GitHub. We'll call the project "Master Project". Any given programmer can use their command line to access any of the files of Master Project using git commands. When they do this, they are essentially making a copy of the files onto their computer. Now they can make changes to the files without affecting Master Project. Once they are finished making their changes, they can do what is called a "commit", which is essentially a submission of their changes that can be reviewed before it is published. Thanks to the version control capabilities of git, any of the programmers working on Master Project can see who has made changes, when they did, and why. That is why these things are so important - any number of programmers can simultaneously work on a huge project with a bunch of files - all without messing each others' work up! This means a project can be finished faster, more thoroughly, and with unique input. It also makes programmers happier and more connected - they can even post a link to their project and ask for help from anyone on the internet who's interested!
